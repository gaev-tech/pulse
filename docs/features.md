# Features

## Auth
- Регистрация и вход по email: пользователь вводит email, на почту отправляется ссылка, по которой он входит в систему авторизованным
- Logout
- Access token: 15 минут, Refresh token: 30 дней
- Несколько активных аккаунтов одновременно, переключение между сессиями без повторного логина; при переключении применяются язык и тема выбранного пользователя

## Teams
- Создание команды (name, prefix)
- Приглашение существующего пользователя по email с выдачей права просмотра задач и последующей возможностью дать остальные права
- Удаление участника с отзывом всех его прав и каскадным удалением всех выданных им прав другим пользователям по всем командным задачам
- Просмотр списка участников с возможностью изменить их права или удалить участника
- Удаление команды

## Permissions
**Права по задаче:** просмотр; редактирование каждого из редактируемых полей по отдельности (название, описание, статус, ответственный, метки, ссылки, связи, блокирование, родитель, вложения); share — право выдавать доступ к задаче другим пользователям. Пользователь не может выдать права, которых у него нет. Цепочка шаринга (A → B → C): если у A забирают право, оно каскадно пропадает у B и C.

**Командные права:** смена владельцев команды; редактирование командных меток; редактирование списка командных меток; редактирование командных фильтров; редактирование командных автоматизаций; все права по задаче.

**Шаринг задачи:** пользователь с правом `share` может выдать другому пользователю любые права по задаче, которые есть у него самого. При шаринге выдаются только права по задаче.

## Tasks
- Создание личной задачи (owner = текущий пользователь)
- Создание командной задачи (owner = команда, требует permission в команде)
- Поля: название, описание (markdown), статус (`opened`/`closed`), ответственный, метки, ссылки, связи, блокирование, родительская задача, дочерние задачи, вложения
- Task key: автоматически генерируется (`#123` для личных, `PREFIX-123` для командных) для удобного использования в фильтрах и API
- Копирование ссылки на задачу, содержащей её ключ

## Filters
- Фильтр бывает личным или командным
- Критерии: автор, ответственный, название (contains), ключ (contains), описание (contains), статус, метки, ссылки, дата создания, дата изменения, наличие связи (в общем или с конкретной задачей), наличие блокирования (в общем или с конкретной задачей), родительская задача, имя вложения (contains). Для личных фильтров дополнительно: команда
- Применение фильтра возвращает список задач, соответствующих критериям
- Настройки таблицы результатов (набор и порядок необязательных колонок, сортировка) сохраняются на бэкенде и применяются для всех пользователей с доступом к фильтру
- Сортировка: клик по заголовку колонки — первичная сортировка; shift+клик по другому заголовку — вторичная сортировка (применяется при равенстве первичной)

## Labels
- Личные метки применяются к личным задачам, командные — к задачам команды
- Метка имеет название и цвет
- При переименовании метки она переименовывается во всех задачах с сохранением события в каждой задаче

## Search
- Поиск по задачам по ключу, названию и описанию (contains)
- Сортировка: сначала полное соответствие ключа или названия, затем по дате последнего открытия задачи пользователем
- Факт открытия задачи фиксируется на бэкенде при открытии Task Screen, Task Modal или получении задачи через API
- Те же правила сортировки используются при выборе блокирующих, связанных, родительских и дочерних задач

## Automations
- Личные и командные автоматизации
- Состав: триггер (обязательно), критерии отбора по полям payload триггера (необязательно), одно или несколько действий — вызов Public API системы или стороннего API с доступом к payload триггера
- Автоматизации можно включать и выключать

## Private Access Token
- Генерируется авторизованным пользователем
- Используется для доступа к Public API и автоматизациям
- Действует до следующей генерации

## Activity Feed
- Личная лента: все события по задачам, доступным пользователю, с учётом окна доступа
- Командная лента: все события по задачам команды
- Лента по фильтру: изменения фильтра, попадание/исчезновение задач из результатов, все события по задаче в период её вхождения в результаты
- Динамическое обновление ленты: при появлении новых событий фронтенд получает сигнал и обновляет данные

## Glossary

| Entity | Description |
|---|---|
| **User** | Зарегистрированный пользователь. Может иметь несколько активных сессий одновременно. |
| **Session** | JWT-пара (access + refresh token) одного пользователя. Хранится в localStorage на клиенте. Несколько сессий = несколько аккаунтов. |
| **Team** | Команда с уникальным `prefix` — используется в URL (`/teams/ACME`), ключах задач (`ACME-123`), поиске и API. |
| **Team Member** | Участник команды. Имеет командные права, определяющие доступ к ресурсам команды. |
| **Task** | Задача. Неизменяемые поля: ключ, владелец (пользователь или команда). Редактируемые поля: название, описание (markdown), статус, ответственный, метки, ссылки, связи, блокирование, родительская задача, вложения. |
| **Task Key** | Уникальный идентификатор задачи: `#123` для личных, `PREFIX-123` для командных. Используется в API, URL, поиске. |
| **Task Sequence** | Счётчик для генерации task key. Отдельный per-user и per-team. Инкремент атомарный, внутри транзакции. |
| **Filter** | Сохранённый набор типизированных критериев для получения списка задач. Бывает личным и командным. |
| **Permission** | Право субъекта (user или team) совершать действие над ресурсом. Два вида: по задаче и командные. Три уровня выдачи: владелец, команды, непосредственные. |
| **Task Share** | Механизм предоставления доступа к задаче конкретному пользователю через выдачу permissions. Требует права `share` у инициатора. |
| **Event** | Запись об изменении в системе. Хранится в append-only таблице `events`. Основа Activity Feed. |
| **Activity Feed** | Лента событий. Три режима: личная, командная, по фильтру. |
| **Private Access Token** | Персональный токен для доступа к Public API и автоматизациям. Генерируется авторизованным пользователем, действует до следующей генерации. |
| **Personal and Team Labels** | Метки для категоризации задач. Имеют название и цвет. Личные — для личных задач, командные — для задач команды. |
| **Task Link** | Ссылка на задачу, содержащая её ключ. При переходе открывает задачу в UI системы. |
| **Related Links** | Список внешних ссылок, прикреплённых к задаче. Каждая имеет URL и опциональное название. |
| **Blocking Tasks** | Задачи, блокирующие выполнение текущей. Цепочка блокирования не может замыкаться — при назначении выполняется валидация. |
| **Related Tasks** | Связанные задачи. Связь двусторонняя: при добавлении отображается у обеих задач. |
| **Status** | Состояние задачи: `opened` или `closed`. |
| **Opened Task** | Задача со статусом `opened`. |
| **Closed Task** | Задача со статусом `closed`. |
| **Description** | Описание задачи в формате markdown. Поддерживает: ссылки, картинки, жирный, курсив, подчёркивание, зачёркивание, списки (нумерованный/ненумерованный с вложенностью), чеклист, заголовки 5 уровней, моноширинный текст, блок кода. |
| **Attachments** | Файлы, прикреплённые к задаче. |
| **Search** | Поиск по задачам по ключу, названию и описанию (contains). Сортировка: полное соответствие → дата последнего открытия пользователем. |
| **Personal and Team Automation** | Автоматизация: триггер + опциональные критерии + одно или несколько действий (вызов API). Можно включать и выключать. |
| **Automation Triggers** | Триггеры для автоматизаций. По задаче, команде, личные и по ленте активности. |
| **Switch Account** | Переключение между авторизованными аккаунтами с применением языка и темы выбранного пользователя. |
