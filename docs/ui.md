# UI

## Navigation Sidebar
Меню навигации в левой части экрана на всю высоту. Сверху вниз: иконка и название системы; кнопка перехода на Personal Activity Screen; раздел "Личные фильтры" (схлопывается, состояние сохраняется на бэкенде) — кнопка создания фильтра, список личных фильтров (клик — Filter Results Screen, в строчке — кнопка открытия Filter Activity Screen); кнопка импорта личных задач (открывает Import Tasks Modal без team_id); раздел "Команды" (схлопывается, состояние сохраняется на бэкенде) — кнопка создания команды, список команд (каждая схлопывается, состояние сохраняется на бэкенде; клик — Team Activity Screen; в строчке — кнопка добавления командного фильтра; в строчке — кнопка импорта задач команды (видна при наличии права `team.import`, открывает Import Tasks Modal с team_id); под строчкой — список фильтров команды: клик — Filter Results Screen, в строчке — кнопка Filter Activity Screen); кнопка открытия модалки личных автоматизаций; кнопка открытия модалки личных меток; кнопка перехода на About Screen; кнопка перехода на Documentation Screen; аватарка и имя пользователя — открывают Profile Popup.

## Profile Popup
Всплывающее меню, открывающееся при клике на аватарку и имя в нижней части Navigation Sidebar. Содержит сверху вниз: переключатель языка (русский/English), переключатель темы (светлая/тёмная/системная), список всех авторизованных на данном клиенте пользователей (клик — переход на Personal Activity Screen выбранного пользователя) с кнопкой "Добавить пользователя" в конце списка (ведёт на Auth Screen), кнопка выхода из системы.

## Auth Screen
Экран авторизации. Содержит форму входа и регистрации.

## About Screen
Экран знакомства с системой: описание продукта, его ценность, маркетинговая информация со скриншотами.

## Documentation Screen
Экран с документацией по системе.

## Personal Activity Screen
Экран личной ленты событий пользователя. Отображает все события по задачам, доступным пользователю, с учётом окна доступа.

## Team Activity Screen
Экран ленты событий команды. Отображает все события по задачам команды. Содержит фильтр по авторам изменений.

## Team Activity Filter By Member
Фильтр на экране ленты команды, позволяющий отображать события только по выбранным авторам изменений.

## Filter Activity Screen
Экран ленты событий по фильтру. Отображает: изменения фильтра, попадание и исчезновение задач из результатов, все события по задачам в период их вхождения в результаты фильтра.

## Filter Results Screen
Экран результатов фильтра. Верхняя панель: название фильтра (редактируется по клику), строка критериев, кнопка настроек таблицы (Table Settings Popup), кнопка "..." (Filter Popup с переходом на Filter Activity Screen и удалением фильтра). При изменении критерия результаты обновляются немедленно, появляется кнопка "Сохранить фильтр".

**Простой режим:** строка критериев содержит инпут поиска (по ключу и названию) и теги ответственного, статуса, меток (для личных фильтров — также тег команды).

**RSQL режим:** строка критериев заменяется на textarea для ввода RSQL-выражения.

Переключение между режимами — в Table Settings Popup. При переключении значения критериев сбрасываются.

Таблица задач: обязательные ячейки — ключ и название; необязательные — метки, команда (только в личных фильтрах), ответственный, статус, родительская задача, наличие вложений/блокеров/связей/ссылок, даты создания и изменения, авторы создания и изменения. Набор и порядок необязательных ячеек настраивается перетаскиванием в Table Settings Popup. Сортировка включается по клику на заголовок колонки (только для отображаемых колонок), направление обозначается стрелкой; shift+клик по другому заголовку добавляет вторичную сортировку — в заголовке отображается стрелка с цифрой «2». Настройки таблицы (набор, порядок, сортировка) сохраняются для фильтра и применяются для всех пользователей с доступом к нему.

## Task Share Modal
Модальное окно шаринга задачи. Поле поиска пользователя и список прав для выдачи. Право `view` отображается задизейблированным — оно выдаётся автоматически и может быть отозвано только через полный отзыв доступа к задаче.

## Task Screen
Полноэкранная страница задачи. Две части: левая (широкая) — название, описание, лента событий (обновляется динамически); правая — элементы управления сверху, затем остальные поля задачи (статус, ответственный, метки, ссылки, связи, блокирование, родитель, вложения).

## Markdown Editor
Редактор описания задачи с тулбаром и поддержкой горячих клавиш. Используется в Task Screen, Task Modal и Task Create Modal.

**Тулбар:** кнопки «Жирный», «Курсив», «Подчёркивание», «Зачёркивание», «Заголовок» (выбор уровня 1–5), «Нумерованный список», «Ненумерованный список», «Чеклист», «Код» (inline), «Блок кода», «Ссылка», «Изображение». Кнопки применяются к выделенному тексту; без выделения — вставляют маркеры в позицию курсора. Повторное нажатие снимает форматирование.

**Горячие клавиши:** Ctrl+B / Cmd+B — жирный, Ctrl+I / Cmd+I — курсив, Ctrl+U / Cmd+U — подчёркивание, Ctrl+Shift+S / Cmd+Shift+S — зачёркивание, Ctrl+Shift+K / Cmd+Shift+K — inline код, Ctrl+K / Cmd+K — вставка ссылки. Применяются к выделению; повторное нажатие снимает форматирование.

**Drag-and-drop и вставка:** перетаскивание изображений и файлов в область редактора; вставка изображений из буфера обмена (Ctrl+V / Cmd+V); вставка текста с сохранением форматирования.

**Поведение списков:** Tab — увеличение вложенности, Shift+Tab — уменьшение; Enter — продолжение списка; двойной Enter на пустом пункте — завершение списка. Чекбоксы в чеклисте переключаются кликом.

**Режимы:** переключение между редактированием и предпросмотром.

## Task Modal
Модальное окно задачи. Отображает те же поля и структуру, что и Task Screen, но поверх текущего экрана. Содержит кнопку "Открыть задачу в отдельной странице".

## Task Create Modal
Модальное окно создания задачи. Позволяет указать название, описание и основные поля. Владелец задачи выбирается из двух вариантов: личная задача или одна из команд, где у пользователя есть право создания задач. После создания закрывается и открывается Task Modal созданной задачи — пользователь остаётся на текущей странице.

## Team Create Modal
Модальное окно создания команды. Поля: название (уникальное), prefix (уникальный). Позволяет сразу добавить участников. В списке прав право `view` отображается задизейблированным — оно обязательно и может быть отозвано только через удаление участника из команды.

## Team Edit Modal
Модальное окно редактирования команды. Позволяет изменить название и prefix, управлять правами участников, а также выдавать права пользователям, которых ещё нет в команде. Содержит кнопку удаления команды. В списке прав право `view` отображается задизейблированным — оно обязательно и может быть отозвано только через удаление участника из команды.

## Filter Create Modal
Модальное окно создания фильтра. Поля: название, тип (личный или командный; при выборе командного — список команд, в которых у пользователя есть право редактирования фильтров), режим фильтрации и критерии.

**Простой режим:** инпут поиска (по ключу и названию) и теги ответственного, статуса, меток. Для личных фильтров дополнительно: тег команды.

**RSQL режим:** textarea для ввода RSQL-выражения.

Переключение между режимами сбрасывает введённые критерии.

## Automation Modal
Модальное окно управления автоматизациями. Содержит список автоматизаций — каждая строка: название, триггер, переключатель включения/выключения, кнопка удаления. Клик по строке (не по переключателю или кнопке) — инлайн-раскрытие с редактированием названия, критериев отбора триггеров и вызовов API. Одновременно может быть раскрыта только одна строка.

## Global Search
Кнопка поиска под заголовком системы в Navigation Sidebar. При нажатии открывает Search Modal.

## Search Modal
Модальное окно поиска. Содержит поле ввода и список результатов под ним.

## Confirmation Popup
Всплывающее окно подтверждения опасной операции. Появляется перед удалением любой сущности и перед удалением участника команды. Содержит текст «Вы подтверждаете <операция>?», красную кнопку «Да, подтверждаю» и кнопку «Отмена».

## Task Link Toast
Всплывающее уведомление о том, что ссылка на задачу скопирована. Появляется после нажатия кнопки "Скопировать ссылку на задачу".

## Blocking Toast
Всплывающее уведомление о том, что задача не может быть заблокирована, так как сама находится в цепочке блокирующих задач.

## Blocking Tasks Popup
Всплывающее окно для просмотра и редактирования списка блокирующих задач. При добавлении — поиск по ключу, названию, описанию (contains). Задачи, образующие цикл блокирования, подсвечиваются восклицательным знаком и недоступны для выбора.

## Related Tasks Popup
Всплывающее окно для просмотра и редактирования списка связанных задач. При добавлении — поиск по ключу, названию, описанию (contains).

## Related Links Popup
Всплывающее окно для просмотра и редактирования списка внешних ссылок задачи.

## Import Tasks Modal
Модальное окно импорта задач. Содержит три таба: CSV, Jira, GitHub.

**Таб CSV:** поле загрузки файла, поле выбора колонки для origin_id (`id_column`). Кнопка "Import".

**Таб Jira:** поля: instance URL, project key, email, API token. Кнопка "Import".

**Таб GitHub:** поля: owner, repo, access token. Кнопка "Import".

**Состояния таба:**
1. **Форма ввода** — заполнение параметров, кнопка "Import".
2. **В процессе** — прогресс (обработано X из Y). Модалку можно закрыть, импорт продолжается на сервере.
3. **Завершён успешно** — результат: "Imported: X, Updated: Y". Две кнопки: "Accept" (сбрасывает форму для нового импорта) и "Restart" (перезапускает с теми же параметрами).
4. **Завершён с ошибками** — результат + раскрываемый список ошибок. Те же кнопки "Accept" / "Restart".

Закрытие модалки не прерывает импорт. При повторном открытии таб показывает текущее состояние (прогресс или результат). Прогресс обновляется динамически через WebSocket. При завершении импорта отображается Import Completed Toast независимо от того, открыта модалка или нет.

## Import Completed Toast
Всплывающее уведомление о завершении импорта. Содержит краткий результат: количество импортированных и обновлённых задач. Автоматически исчезает.

## Validation

- Валидация выполняется при потере фокуса полем (on blur) и при попытке отправить форму (on submit)
- Невалидное поле подсвечивается красной рамкой
- Под полем отображается текст ошибки
- Кнопка отправки формы остаётся активной — ошибки показываются только после попытки отправки или потери фокуса, а не заблаговременно
- При серверной ошибке валидации (400) — ошибка отображается под соответствующим полем; если поле не определено — отображается Error Toast
- При нажатии кнопки, выполняющей запрос, на кнопке отображается лоадер на время выполнения запроса
- Если кнопка находится в модалке или попапе — модалка/попап не закрываются до успешного завершения или ошибки; исключение — кнопка запуска импорта, так как импорт является фоновой операцией

## Zero State
Если в разделе, списке или таблице нет данных для отображения (нет задач, нет фильтров, нет участников, нет событий и т.д.), вместо пустого пространства отображается zero-state — сообщение об отсутствии данных.

## Error Toast
Тост с сообщением об ошибке. Появляется при неуспешном выполнении любого запроса к бэкенду. Автоматически исчезает.

## Unsaved Changes Confirmation Modal
Модальное окно подтверждения потери несохранённых данных. Появляется при попытке закрыть модальное окно с несохранёнными изменениями. Содержит текст «Вы уверены? Несохранённые данные будут потеряны», кнопку «Да, закрыть» и кнопку «Отмена».

## 403 Screen
Экран отсутствия доступа. Отображается при открытии задачи, к которой у пользователя нет доступа.

## 404 Screen
Экран «Не найдено». Отображается при открытии несуществующей задачи, несуществующего или недоступного фильтра, несуществующей или недоступной команды.
