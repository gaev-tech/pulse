# TC-195: RSQL: оператор =in= (входит в список)

**Предусловие:** Пользователь авторизован. Существует фильтр в RSQL режиме. Открыт Filter Results Screen. Существуют задачи: #1 — ответственный "Иванов" (uuid1), #2 — ответственный "Петров" (uuid2), #3 — ответственный "Сидоров" (uuid3).

**Шаги:**
1. В строке критериев (textarea) ввести выражение `assignee_id=in=(uuid1,uuid2)`, подставив реальные UUID пользователей "Иванов" и "Петров".
2. Дождаться обновления результатов.

**Ожидаемый результат:**
- В таблице результатов отображаются задачи #1 и #2.
- Задача #3 с ответственным "Сидоров" не отображается.
- Оператор `=in=` корректно фильтрует задачи, значение которых входит в указанный список.
