# TC-360: Responsive permission check — поля задачи доступны/недоступны в зависимости от прав

**Предусловие:** Пользователь авторизован. Существует командная задача `ACME-22`. Пользователь имеет права: `view`, `edit.title`, `edit.status`. Пользователь не имеет прав: `edit.description`, `edit.assignee`, `edit.labels`, `edit.links`, `edit.relations`, `edit.blocking`, `edit.parent`, `edit.attachments`, `share`.

**Шаги:**
1. Открыть задачу `ACME-22` в Task Screen или Task Modal.
2. Попытаться отредактировать название задачи.
3. Попытаться изменить статус задачи.
4. Попытаться отредактировать описание задачи.
5. Попытаться изменить ответственного.

**Ожидаемый результат:**
- Название задачи доступно для редактирования (есть право `edit.title`) — поле активно, можно вносить изменения.
- Статус задачи доступен для изменения (есть право `edit.status`) — переключатель активен.
- Описание задачи недоступно для редактирования (нет права `edit.description`) — поле заблокировано или отсутствует элемент управления редактированием.
- Ответственный недоступен для изменения (нет права `edit.assignee`) — элемент управления заблокирован.
- Поля без соответствующих прав отображаются в режиме только для чтения.
